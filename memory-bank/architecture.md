## 2026-04-05 第二次真实浏览器验收补充架构洞察

- 当前第 29 步最终收口的真实验收方式不是复用固定 `8000/4173` 端口，而是“独立 PostgreSQL 数据库 + 独立后端端口 + 独立前端端口 + 独立 Worker 进程”的隔离运行方式。这样可以避免本机其他项目占用默认端口时污染验收结论。
- 文档上传链路的关键一致性要求已经进一步被真实浏览器验收证明：上传接口必须先提交数据库事务，再入队 RQ 任务。否则 Worker 抢先消费时会因为读不到 `document/task` 记录而出现 `document_not_found`，并导致 UI 长时间停留在 `UPLOADED`。
- 非法文件上传的真实 UI 行为已经确认：后端返回 `400 unsupported_file_type` 后，前端会在 `DocumentManagerPanel.vue` 中展示“上传失败 / 不支持的文件类型”。此前失败不是功能缺失，而是浏览器脚本把错误提示出现前后的异步异常处理当成主失败信号。
- 删除文档的最终验收标准已经收敛为两层：
  - 前端层：文档列表中的对应卡片消失
  - 后端层：`GET /api/documents/{document_id}` 返回 `404`
  不能再使用“整页文本里是否还出现文档名”作为唯一判断，因为聊天引用区会保留历史引用文本。
